## 사용 방식

각 phase 문서는 아래 순서로 읽고 갱신한다.

1. 이번 단계의 목표를 확인한다.
2. 코드에서 확인할 포인트를 본다.
3. 직접 실행하거나 수정해 본다.
4. 배운 점과 다음 질문을 기록한다.

## 현재 범위

현재 예제는 아래 개념만 다룬다.

- `severity_logger`
- 공통 attribute 등록

- 파일 출력

개념 정리는 먼저 [Concept-01-Core-Sinks-Attributes-And-Helpers.md](./Concept-01-Core-Sinks-Attributes-And-Helpers.md)를 읽고, 이후 phase 문서를 따라가는 순서를 권장한다.
